- verifyAccessToken(token): verify and return payload

- hashPassword(password): bcrypt.hash with cost factor 10
- verifyPassword(password, hash): bcrypt.compare

- generateInviteCode(length=8): cryptographically secure random
- validateInviteCode(code): check exists, not expired, not used (if one-time)

  <action>
Mount all D-XX endpoints under /api/v1/auth/:

POST /register/new-family (D-30):
- Validate with registerNewFamilySchema
- Create family, create user with passwordHash
- Generate tokens, store refreshToken in DB
- Set HttpOnly cookie with accessToken

POST /register/join-family (D-31):
- Validate with registerJoinFamilySchema
- Validate invite code (not expired, not used if one-time)
- Create user with existing familyId
- Mark one-time code as used
- Generate tokens, store refresh
- Set HttpOnly cookie

POST /login (D-08):
- Validate with loginSchema
- Find user, verify password
- Generate tokens, set cookie

POST /logout (D-09):
- Verify token, delete refreshToken from DB
- Clear accessToken cookie

POST /refresh (D-10):
- Verify refresh token in DB (not revoked)
- Optional: rotate (delete old, create new)
- Generate new accessToken, set cookie

POST /family/invite (D-11):
- Authenticate
- Generate invite code (custom or auto)
- Save to FamilyInvite with expiry (7d), oneTimeUse

DELETE /family/invite/:code (D-12):
- Authenticate
- Soft-delete or hard-delete invite code
