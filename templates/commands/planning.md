# Planning Workflow

## Purpose

This procedure defines how to create and maintain project plans using `templates/plan-template.md`.

## Principles

- **Incremental Delivery** - Plans break work into verifiable phases
- **Testable Assertions** - Goals have measurable acceptance criteria
- **Explicit Dependencies** - All dependencies are declared upfront

## Procedure

### 1. Initiate

1. Copy `templates/plan-template.md` to your working document
2. Fill in project name and summary
3. Identify primary goals

### 2. Structure Phases

1. Break work into 2-4 phases maximum
2. Each phase should deliver verifiable value
3. Tasks within phases should be parallelizable where possible

### 3. Define Tasks

For each task:

1. Clear description
2. Concrete deliverable
3. Verification method

### 4. Document Dependencies

1. List all external dependencies with versions
2. Identify internal task dependencies
3. Note constraints that affect delivery

### 5. Review

Constitution Check:

- [ ] All 6 principles addressed
- [ ] Work is incrementally deliverable
- [ ] Each goal is measurable
- [ ] Dependencies explicitly declared

## Verification

- Plan can be explained in one presentation
- Each phase delivers independent value
- All dependencies have version constraints
